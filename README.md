# Word漏洞报告转换器

将 H3C 漏洞扫描 Word 报告批量转换为结构化 Excel 表格的桌面工具。

## 功能特性

- 支持批量导入 `.docx` 格式的漏洞报告文件
- 支持按文件夹批量添加
- 自动从文件名中提取 IP 地址（格式：`192.168.1.1-xxx.docx`）
- 仅提取**高风险**和**中风险**漏洞，过滤低风险条目
- 生成 Excel 文件，包含：
  - 漏洞汇总页（所有 IP 汇总，按高风险数量降序排列）
  - 各 IP 独立分页
- 不同 IP 以不同背景色区分，便于阅读
- 转换过程实时日志输出

## 环境要求

- Python 3.7+
- 依赖库：

```bash
pip install python-docx openpyxl
```

## 使用方法

```bash
python H3C_word-exchange.py
```

启动后：

1. 点击「添加文件」选择单个或多个 `.docx` 文件，或点击「添加文件夹」批量导入
2. 点击「浏览」选择输出的 Excel 文件保存路径
3. 点击「开始转换」，等待处理完成
4. 转换日志区域会实时显示每个文件的处理状态

## Word 文档格式要求

工具依赖以下文档结构进行解析：

- 文档中需包含至少两处「主机漏洞详情」段落标题
- 文档中需包含「Web漏洞信息」段落标题作为解析终止标记
- 漏洞信息以表格形式存储，表格中包含以下字段（左列为字段名，右列为内容）：
  - `漏洞名称`
  - `风险等级`（含"高风险"或"中风险"字样）
  - `解决办法`

## 输出 Excel 结构

| 列名   | 说明                     |
|--------|--------------------------|
| IP地址 | 从文件名解析的 IP 地址   |
| 漏洞名称 | 漏洞标题               |
| 风险等级 | 高风险 / 中风险         |
| 解决办法 | 修复建议               |

- 汇总页中各 IP 按**高风险漏洞数量降序**排列
- 每个 IP 拥有独立的 Sheet 页，Sheet 名称为 IP 地址
- 不同 IP 行使用不同背景色区分

## 文件命名规范

Word 文件名需以 IP 地址开头，格式如下：

```
192.168.1.100-漏洞报告.docx
10.0.0.1-扫描结果.docx
```

若文件名不符合规范，IP 地址将显示为 `未知IP_序号`。

## 注意事项

- 仅支持 `.docx` 格式，不支持旧版 `.doc`
- 搭配format-exchange使用
- Excel Sheet 名称最长 31 个字符，超长 IP 会自动截断
- 转换在后台线程执行，界面不会卡死
